# Visionary 模型功能与提示词手册 (工程交互版 - V1.1)

本手册是系统功能实现的核心依据。包含提示词模板、积分档位、以及 UI 交互逻辑。

---

## ⚖️ 档次与交互矩阵 (UI/UX Matrix)

| 模式 | 积分 | UI 预填/引导文本 | 必须上传图片 | 核心能力 |
| :--- | :--- | :--- | :--- | :--- |
| **初级版** | 5 | `主题：【在此输入你想生成的画面】` | 否 | 原生语义、基础写实 |
| **中级版** | 10 | 根据具体模板动态变化 (见下文) | 部分需要 | 风格注入、标准排版 |
| **高级版** | 15 | 根据具体模板动态变化 (见下文) | 部分需要 | 深度叙事、逻辑构图 |

---

## 🟢 初级版 (5 积分)

### 1. 标准版-极致写实
*   **UI 预填**：`主题：【在此输入你想生成的真实画面】`
*   **指令内容**：主题为【主题】，要求仿真现实。

### 2. 电商白底主图
*   **UI 预填**：`产品名称：【在此输入产品名称，如：陶瓷咖啡杯】`
*   **指令内容**：【产品名称】桌面摆件，做成白底电商主图，重新打光，产品更精致，纯白背景，柔光效果，突出产品主体。

### 3. 深色科技感海报
*   **UI 预填**：`海报主题：【在此输入科技主题，如：量子计算 / 脑机接口】`
*   **指令内容**：【主题】封面海报，深色调，信息量大，有科技感，色彩克制，信息主次清晰。

---

## 🟡 中级版 (10 积分)

### 1. 设计类模板
*   **旅游海报**
    *   **UI 预填**：`城市名称：【在此输入城市，如：重庆】`
    *   **指令内容**：全量模版：竖版长图，【城市/地点】旅游主题，包含【景点1】、【景点2】等6个经典景点，每个模块有精致插画+狂草诗句+文字简介，高端杂志排版。
*   **现代室内设计**
    *   **UI 预填**：`装修风格：【在此输入风格，如：法式复古 / 侘寂风】`
    *   **图片约束**：**是 (必须上传原始空间图)**
    *   **指令内容**：高端室内设计图，【风格名称】风，平面图+3D渲染图组合，空间通透，落地窗。
*   **直播截图**
    *   **UI 预填**：`直播内容：【在此输入直播内容，如：大牌美妆专场】`
    *   **指令内容**：全量模版：生成【平台名称】直播截图，【主播/内容描述】，在线人数【人数】，界面真实可辨。

---

## 🔴 高级版 (15 积分)

### 1. 绘梨衣·侧脸叙事海报 (Silhouette Pro)
*   **UI 预填**：`叙事主题：【在此输入主题，如：丝绸之路的繁荣 / 废墟中盛开的玫瑰】`
*   **指令内容**：根据【主题：【主题内容】】自动生成一张收藏版史诗叙事海报：巨大优雅的人物侧脸剪影作为外轮廓，剪影内部自动生长出最契合该主题的完整世界观、标志性场景、角色关系、象征符号。带有双重曝光式联想，偏电影海报与梦幻水彩插画融合风格。画面中需自然加入专属签名“【落款内容】”。

### 2. S型丝绸双重曝光海报
*   **UI 预填**：`宣传城市/主题：【在此输入名称，如：千年古都西安】`
*   **指令内容**：生成一张充满【节日名称】氛围的【年份】城市宣传海报。构图延续 S 型流动感；右下角微缩人物舞动红绸，红绸奇幻变形成壮丽山脉河流，叠加【城市名称】城市手绘图。

### 3. 复古数码人像纪实 (iPhone 5s Style)
*   **UI 预填**：`拍摄地点：【在此输入具体地点，如：香港旺角街头 / 台北宁夏夜市】`
*   **图片约束**：**是 (必须上传人物参考图)**
*   **指令内容**：使用用户照片作为参考，保留人物真实五官特征。场景设定在【地点描述】，风格模拟 iPhone 5s 拍摄的复古数码质感：轻微数字噪点，略微褪色的色彩，亮部轻微过曝。

### 4. 人物关系图海报
*   **UI 预填**：`作品/事件名称：【在此输入，如：三体 / 哈利波特】`
*   **指令内容**：全量模版：生成一张高设计感的人物关系图海报。识别核心设定，提取 6—15 位关键人物，展示血缘、盟友、敌对关系；自动选择构图方式。

### 5. 科普百科图
*   **UI 预填**：`科普对象：【在此输入，如：深海大王乌贼】`
*   **指令内容**：全量模版：生成一张高质量竖版「科普百科图」。包含主视觉、局部放大、圆角信息分区、可视化评分等，模拟高传播度的知识卡片。

---

> [!IMPORTANT]
> **代码实现指引**：
> 1.  **前端交互**：用户点击特定 Style 时，React 状态 `prompt` 必须立即更新为对应的 `UI 预填文本`，方便用户修改。
> 2.  **后端包装**：后端 API 接收到 `prompt` 后，需检测其所属风格，并用全量长指令包裹用户输入的关键词后再发送至 AI。
