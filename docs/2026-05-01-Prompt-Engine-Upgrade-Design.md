# GPT-Image-2 提示词引擎深度升级设计文档 (2026-05-01)

## 1. 背景与目标
当前后端提示词模板过于简陋，且缺乏对“人像仿真”和“环境融合”的深度约束，导致生成结果在高级审美和写实度上无法满足用户需求。本设计旨在将 `docs/功能提示词大全.md` 中的深度指令全量同步至后端，并注入专项“人像仿真补丁”。

## 2. 核心架构设计

### 2.1 模板映射字典 (STYLE_PROMPT_TEMPLATES)
将前端 12 种风格 ID 与文档级深度指令进行 1:1 绑定，使用 `【主题】` 作为动态注入占位符。

### 2.2 人像仿真补丁 (Portrait Fusion Patch)
针对人像类风格（`real`, `vintage_5s`, `live_stream`, `anime`, `eri_silhouette`），在提示词末尾强制追加：
> “画面要求：整体光影必须统一，主体与环境之间有自然的阴影遮蔽和反光交互，严禁背景突兀，确保主体与背景深度融合。”

## 4. 逻辑处理流程
1.  **占位符替换**：提取用户原始输入（如 `主题：小猫`），替换模板中的 `【主题】`。
2.  **补丁注入**：判断 `style_id`，若是人像类则追加“仿真融合”补丁。
3.  **画质增强**：根据任务档次（HD/Master）追加 `masterpiece`, `8k` 等技术后缀。

## 5. 验收标准
- [ ] 所有前端风格均有对应的高级中文提示词。
- [ ] 人像生成结果中，人物与背景的光影关系自然，无明显拼贴感。
- [ ] 复杂任务（如百科、关系图）生成的排版逻辑符合文档描述。
