# Visionary 部署指南

**最后更新**: 2026-05-06（基于代码实测）

---

## 1. 本地开发环境

### 1.1 前提条件

- Python ≥ 3.10
- Node.js ≥ 18（生产构建需 v20+，Vite 6 要求）
- MySQL 5.7+（远程或本地）
- SSH 免密登录服务器（如需连接远程数据库）

### 1.2 快速启动

```bash
# 1. 安装 Python 依赖
pip install -r requirements.txt

# 2. 安装前端依赖
cd frontend && npm install && cd ..

# 3. 配置环境变量
cp .env.example .env
# 编辑 .env：填写 DATABASE_URL, SECRET_KEY 等

# 4. 配置 SSH 免密登录（连接远程数据库时必须）
ssh-copy-id root@<服务器IP>

# 5. 一键启动
chmod +x start.sh
./start.sh
```

`start.sh` 自动执行：
1. 清理旧进程（8000、5173、3307 端口）
2. 启动 SSH 隧道守护进程（每 5 秒检测，断线自动重连）
3. 等待数据库就绪后启动 FastAPI（`uvicorn api.index:app --port 8000`）
4. 启动 Vite 前端开发服务器（`npm run dev --port 5173`）

### 1.3 数据库初始化

```bash
# 导入初始表结构
mysql -h 127.0.0.1 -P 3307 -u <user> -p <database> < sql/init.sql

# 应用增量补丁（按版本顺序）
mysql -h 127.0.0.1 -P 3307 -u <user> -p <database> < sql/patch_v1.1_configs.sql
mysql -h 127.0.0.1 -P 3307 -u <user> -p <database> < sql/patch_v1.2_feedback.sql
```

## 2. 生产环境部署

### 2.1 架构拓扑

```
用户浏览器
  │
  ├─ 静态资源 (SPA)  →  Nginx :80  →  frontend/dist/
  │
  └─ API 请求  →  Nginx proxy_pass  →  FastAPI :8000
                                          │
                                          ├─ MySQL (本地 :3306 或远程隧道)
                                          └─ 腾讯云 COS
```

### 2.2 前端部署（Vercel）

```bash
# 根目录 package.json 已配置 monorepo 构建脚本
npm run build
# 产物自动输出至 public/ 目录，Vercel 读取此目录作为静态站点根
```

Vercel 项目设置：
- **Framework**: Other
- **Build Command**: `npm run build`
- **Output Directory**: `public`
- **Root Directory**: `./`

### 2.3 后端部署（腾讯云 + 宝塔面板）

#### Node.js 环境升级（Vite 6 要求 ≥ v20）

宝塔面板 → 软件商店 → Node.js 版本管理器 → 安装 v20.18.3 LTS。

#### 代码同步

```bash
cd /www/wwwroot/gpt-image-api
git fetch --all
git reset --hard origin/main
```

#### 前端构建（服务器端）

```bash
cd /www/wwwroot/gpt-image-api/frontend
rm -rf node_modules package-lock.json  # 环境变动后必须清理
npm install
npm run build
# 产物在 frontend/dist/
```

> 若构建报 `ENOTDIR: .user.ini`：宝塔防跨站文件具有不可变属性，需 `chattr -i frontend/dist/.user.ini && rm -rf frontend/dist`。

#### 后端启动

```bash
chmod +x server_start.sh
./server_start.sh
```

`server_start.sh` 自动清理 8000 端口旧进程，启动 `uvicorn api.index:app --host 0.0.0.0 --port 8000`。

#### Nginx 配置（宝塔站点配置）

```nginx
server {
    listen 80;
    server_name <你的域名或IP>;
    root /www/wwwroot/gpt-image-api/frontend/dist/;
    index index.html;

    # SPA 路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api {
        proxy_pass http://127.0.0.1:8000;
        proxy_connect_timeout 300s;
        proxy_read_timeout 300s;        # 生图长任务需要 5 分钟
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 2.4 数据库连接策略

| 场景 | 方案 | 连接地址 |
|:---|:---|:---|
| **服务器后端 → 本地 MySQL** | 直连 | `127.0.0.1:3306` |
| **本地开发 → 远程 MySQL** | SSH 隧道 | `127.0.0.1:3307` |
| **紧急远程直连** | IP 白名单 | 在腾讯云安全组添加本地公网 IP |

> 生产环境数据库**严禁** `0.0.0.0/0` 全放行。本地开发首选 SSH 隧道而非白名单（家庭宽带 IP 会变动）。

## 3. 环境变量与动态配置

### 3.1 本地 `.env`

仅需配置**数据库连接**和**本地开发专用**的环境变量。生产密钥存储在 `system_configs` 表中。

### 3.2 生产动态配置

所有密钥和业务参数通过 `system_configs` 表管理，由 `backend/core/config.py` 的 `get_config(key)` 读取。**修改后 60 秒内自动生效，无需重启服务**。

```sql
-- 修改 API 密钥示例
UPDATE system_configs SET config_value = 'sk-new-key' WHERE config_key = 'OPENAI_API_KEY';
-- 修改支付配置
UPDATE system_configs SET config_value = 'https://new-api.zhifule.com' WHERE config_key = 'PAYLE_API_URL';
```

完整配置项列表见 [数据库字段详解](../技术规范/数据库字段详解.md) 第 4 节。

## 4. 首次部署检查清单

- [ ] Node.js 版本 ≥ 20（`node -v`）
- [ ] Python 版本 ≥ 3.10（`python3 --version`）
- [ ] 数据库已导入 `sql/init.sql` 及所有补丁脚本
- [ ] `system_configs` 表已填充全部生产密钥
- [ ] Nginx `proxy_read_timeout` ≥ 300s
- [ ] Nginx `root` 指向 `frontend/dist/`
- [ ] COS 存储桶已创建且密钥配置正确
- [ ] 支付乐回调 URL 外网可达
- [ ] 宝塔面板已配置数据库定期备份

## 5. 常见部署故障

| 症状 | 原因 | 解决 |
|:---|:---|:---|
| 前端构建报 `CustomEvent is not defined` | Node.js < v20 | 宝塔安装 v20.18.3 |
| 前端构建报 `Cannot find native binding` | Node 版本变动后旧依赖残留 | `rm -rf node_modules && npm install` |
| 构建报 `.user.ini` 权限错误 | 宝塔防跨站文件不可变 | `chattr -i dist/.user.ini && rm -rf dist` |
| API 返回 503 | 数据库连接断开 | `lsof -i:3307` 检查隧道 |
| 页面改了代码不生效 | Nginx 缓存或浏览器缓存 | Cmd+Shift+R 强刷；检查 Nginx root 路径 |
| `git pull` 冲突 | 服务器有未跟踪文件 | `git reset --hard origin/main` |
| 部署后 `.sh` 脚本无权限 | Git 不保留执行位 | `chmod +x *.sh` |
