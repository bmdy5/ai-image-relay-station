# 服务器维护手册

**最后更新**: 2026-05-06（基于代码审计）

---

## 1. 启动方式

**标准启动（推荐）**：
```bash
./start.sh
```
`start.sh` 自动执行：
1. 清理旧的 8000（后端）、5173（前端）、3307（隧道）端口进程。
2. 启动 SSH 隧道守护循环（每 5 秒检测一次 3307 端口，断线自动重连）。
3. 等待隧道就绪后，启动 FastAPI 后端（`uvicorn api.index:app --port 8000`）。
4. 启动 Vite 前端开发服务器（`npm run dev`）。

**服务器 IP**: 从 `system_configs` 表中的 `SERVER_IP` 获取。

---

## 2. SSH 数据库隧道

项目通过本地 `3307` 端口映射访问远程 MySQL（远程端口 3306）。

```bash
# 核心命令
ssh -N -L 3307:localhost:3306 root@{SERVER_IP}

# 检查隧道状态
lsof -i:3307

# 手动强制重启隧道（先杀旧进程）
PID=$(lsof -t -i:3307) && [ -n "$PID" ] && kill -9 $PID
ssh -N -L 3307:localhost:3306 root@{SERVER_IP} &
```

**前提条件**：部署机必须已配置 SSH 免密登录（`ssh-copy-id root@{SERVER_IP}`）。

## 3. 日志诊断

| 日志文件 | 监控内容 | 命令 |
| :--- | :--- | :--- |
| `tunnel.log` | SSH 隧道断连与重连记录 | `tail -f tunnel.log` |
| `backend.log` | FastAPI 请求日志、API 调用错误、[Task Audit] 审计报告 | `tail -f backend.log` |
| `frontend.log` | Vite 构建错误 | `tail -f frontend.log` |

**backend.log 重要标记**：
- `[Task Audit] ID:xxx SUCCESS`：生图成功，包含收益/成本/耗时信息。
- `[Task Error] ID:xxx`：生图失败，含原因。
- `[Config Error]`：数据库配置读取失败，系统已降级到 `.env`。
- `[Invitation Reward]`：邀请奖励发放记录。

## 4. 系统动态配置（免重启热更新）

所有密钥存储在 `system_configs` 表中，通过后台界面或直接 SQL 修改即可，**60 秒内自动生效**，无需重启服务。

**修改示例（直连数据库）**：
```sql
UPDATE system_configs 
SET config_value='sk-new-key-xxx' 
WHERE config_key='OPENAI_API_KEY';
```

## 5. 常见故障排查

| 症状 | 排查步骤 |
| :--- | :--- |
| API 返回 500，日志显示数据库错误 | 检查 `lsof -i:3307`，隧道是否存活 |
| 生图一直 pending，不出结果 | 查 `backend.log` 是否有 `[Task Error]`；检查 `OPENAI_API_KEY` 是否有效 |
| 用户积分卡死（frozen_points 不释放） | 用户点击"重置任务"按钮，或管理员后台手动操作 |
| 支付回调不触发到账 | 检查 `PAYLE_NOTIFY_URL` 是否为外网可达地址；查看支付乐后台回调日志 |
| 邮件验证码发送失败 | 检查 SMTP 相关配置（存储在 `system_configs` 中） |
