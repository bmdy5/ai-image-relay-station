# 架构概览

**最后更新**: 2026-05-06（基于代码审计）

---

## 1. 技术栈（完整）

| 层级 | 技术 | 版本要求 |
| :--- | :--- | :--- |
| **前端** | React 19 + Vite 8 + Vanilla CSS | Node.js ≥ 18 |
| **后端** | FastAPI + Uvicorn + SQLAlchemy | Python ≥ 3.10 |
| **数据库** | MySQL（通过 PyMySQL 连接） | MySQL 5.7+ |
| **对象存储** | 腾讯云 COS | ap-guangzhou |
| **AI 模型** | OpenAI gpt-image-2（通过中转站）| — |
| **提示词润色** | 阿里云通义千问（DashScope） | — |
| **支付** | 支付乐（Zhifule）| MD5 签名 |
| **邮件** | SMTP（用于邮箱验证码） | — |

## 2. 请求链路（完整数据流）

```
用户浏览器
  │
  ├─ GET  静态资源  →  Vercel / Nginx (frontend/dist)
  │
  └─ API 请求  →  FastAPI (Port 8000)
                      │
                      ├─ JWT 鉴权  (backend/core/security.py)
                      │
                      ├─ 生图任务
                      │    ├─ 1. 预扣 frozen_points（MySQL）
                      │    ├─ 2. 后台任务 BackgroundTasks 异步执行
                      │    ├─ 3. 提示词 wrap_prompt 包装
                      │    ├─ 4. 调用 gpt-image-2 API（中转站）
                      │    ├─ 5. 图片转存至腾讯云 COS
                      │    └─ 6. 更新 image_logs，扣减 points
                      │
                      └─ MySQL（通过 SSH 隧道 3307→3306）
```

---

## 3. 动态配置机制（重要）

后端所有密钥和配置通过 `backend/core/config.py` 的 `get_config(key)` 读取：

1. 优先读取 `system_configs` 数据库表（内存缓存，**60 秒 TTL**）。
2. 若数据库中无此项或值为空，降级读取 `.env` 环境变量。
3. **修改数据库配置后无需重启**，60 秒内自动生效。

---

## 4. 并发控制（任务锁）

- 每用户最多 **3 个** 并发任务（`active_count >= 3` 则 HTTP 429）。
- 超时机制：httpx 客户端 `timeout=180s`（3 分钟），超时视为失败，`frozen_points` 自动释放。
- 用户可通过 `POST /image/reset` 手动清理卡死的并发锁。

## 5. 项目目录结构

```
.
├── api/              # FastAPI 入口（index.py 挂载所有路由）
├── backend/
│   ├── api/          # 路由层（auth, image, payment, user, admin, feedback）
│   ├── core/         # 核心工具（config, security, cos, email, payment, deps, prompt）
│   ├── crud/         # 数据访问层（image, user, recharge）
│   ├── models/       # ORM 模型（models.py）和数据库连接（database.py）
│   ├── schemas/      # Pydantic 校验 Schema
│   └── services/     # 业务逻辑层（auth_service, image_service）
├── frontend/         # React + Vite 前端工程
├── sql/              # 初始化 SQL 和补丁脚本
├── scripts/          # 工具脚本（如 migrate_uid.py）
├── start.sh          # 本地一键启动（含 SSH 隧道守护）
└── .env              # 本地环境变量（数据库 URL 必须保留）
```
