# 产品需求文档 (PRD)

**版本**: V2.1（基于代码实测修正）
**最后更新**: 2026-05-06

---

## 1. 产品定位

Visionary 是一个基于 GPT-Image 2（gpt-image-2 模型）的 AI 绘图中转平台。核心解决了国内用户访问 OpenAI 的网络门槛，并提供了比官方更灵活的积分计费模式和丰富的风格模板。

---

## 2. 注册与用户体系

### 2.1 注册方式（两种，均已上线）
- **邮箱注册**：需邮箱验证码（6位，5分钟有效），邮箱和用户名均唯一。
- **手机号注册**：11 位手机号直接注册，不需要短信验证码，用户名默认为 `user_后四位`。

### 2.2 内测限额
当前处于**内测阶段**，系统硬限制注册总人数 **≤ 100 人**，超出则返回 `内测名额已满`。

### 2.3 邀请体系（已上线）
- 邀请码 = 用户的 `uid` 字段。
- 受邀者注册时填写邀请码，**注册即获得 +5 积分**（`INVITE_JOIN_*` 流水）。
- 受邀者**完成首次成功生图后**，邀请人获得 **+10 积分**（`INVITE_REWARD_*` 流水）。
- 邀请人每日收到此类奖励上限为 **5 次**，同一受邀者只奖励一次。

### 2.4 特殊积分奖励（已上线）
- **PWA 安装奖励**：用户安装桌面端 PWA 后，调用 `/auth/claim-install-reward`，获得 **+10 积分**（每人仅限一次）。

## 3. 生图功能

### 3.1 三档画质（代码实测）

| 档位名 | 积分消耗 | 迭代上限 | 提示词增强 |
| :--- | :--- | :--- | :--- |
| `standard` | **5 积分** | 不支持迭代 | 无额外增强 |
| `hd` | **10 积分** | 最多 2 次变体 | 追加 `high quality, 4k, sharp focus` |
| `master` | **15 积分** | 最多 3 次变体 | 追加 `masterpiece, ultra-high definition, 8k, unreal engine 5 render, cinematic lighting` |

> 内部均调用 `gpt-image-2` 模型，`quality` 参数统一设为 `low` 以控制成本。**分辨率由宽高比决定**（1:1→1024x1024, 9:16→1024x1536, 16:9→1536x1024），与档位无关。画质差异通过提示词增强和迭代次数体现。

### 3.2 并发限制
同一用户同时最多运行 **3 个** 活动任务（代码：`active_count >= 3` 则拒绝）。

### 3.3 预扣费与冻结积分
- 任务发起时，将积分写入 `frozen_points` 字段（静默预占）。
- 任务成功后，才从 `points` 中真正扣除，并将 `frozen_points` 还原。
- **任务失败时**：仅释放 `frozen_points`，`points` 不变（即失败不扣费）。

### 3.4 提示词风格系统（已上线）
系统内置 20 种风格模板，包括：`real`（极致写实）、`travel_guide`（旅行手账）、`ccd_snap`（CCD 闪拍）、`encyclopedia`（博物馆图鉴）、`relation_map`（人物关系图）等，由 `wrap_prompt` 引擎自动包装。

### 3.5 图生图（已上线）
上传参考图 URL 或 Base64，系统自动切换至 `/v1/images/edits` 端点，通过 **Binary Multipart Stream** 协议直接传送图片字节流。参考图经 Pillow 预处理（1:1 中心裁剪 + 1024x1024 LANCZOS 缩放），确保 API 合规性。

## 4. 支付与充值

### 4.1 在线支付（支付乐 Zhifule，已上线）
- 支持微信支付（`wxpay`），创建订单后跳转支付乐。
- 支付成功后，支付乐异步回调 `/payment/notify`，系统签名验证后**自动到账**，无需人工。

### 4.2 充值套餐

| 充值金额 | 获得积分 | 特殊条件 |
| :--- | :--- | :--- |
| **¥1.00** | **20 积分** | 1 元特惠包，每人仅限一次（再购按 ×10 计算） |
| **¥10.00** | **150 积分** | — |
| **¥30.00** | **500 积分** | — |
| **¥50.00** | **800 积分** | — |
| 自定义 | 金额 × 10 | 如 ¥5 → 50 积分 |

### 4.3 人工报备充值（已上线）
用户也可以通过 `/user/recharge/apply` 上传支付截图，等待管理员在后台通过 `/admin/recharge/audit/{id}` 手动审核放款。
