# 数据库字段详解

本手册基于远程数据库实时探测 + 后端 `backend/models/models.py` 双重核实编写。

---

## 1. `users` (用户表)

| 字段名 | 类型 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | INT | AI | 主键 |
| `username` | VARCHAR(50) | NULL | 用户名，可为空（手机号注册时自动生成 `user_后四位`） |
| `email` | VARCHAR(255) | NULL | 邮箱，唯一索引，用于邮箱注册和找回密码 |
| `phone` | VARCHAR(20) | NULL | 手机号，唯一索引，11位数字 |
| `password_hash` | VARCHAR(255) | - | Bcrypt 加密后的密码哈希 |
| `fingerprint` | VARCHAR | NULL | 浏览器指纹，用于防小号 |
| `uid` | VARCHAR(20) | 随机生成 | 用户的唯一邀请码，用于邀请链接 |
| `points` | INT | 10 | 当前可用积分（不含冻结部分） |
| `frozen_points` | INT | 0 | 生图任务进行中被预扣锁定的积分 |
| `last_ip` | VARCHAR | NULL | 最后一次登录 IP |
| `is_admin` | TINYINT(1) | 0 | 1 = 管理员 |
| `has_used_experience` | TINYINT(1) | 0 | 是否已使用过 1 元特惠体验包（每人仅限一次） |
| `has_install_reward` | TINYINT(1) | 0 | 是否已领取 PWA 安装奖励（10 积分，仅限一次） |
| `last_daily_reward` | DATETIME | NULL | 最近一次每日签到时间（用于判断是否可再次签到） |
| `invited_by_id` | INT | NULL | 邀请人的 `users.id`，外键关联 |
| `created_at` | DATETIME | 当前时间 | 注册时间（北京时间 UTC+8） |

---

## 2. `image_logs` (生图记录表)

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `user_id` | INT | 外键 → users.id |
| `prompt` | TEXT | 用户原始输入的提示词 |
| `final_prompt` | TEXT | 经 `wrap_prompt` 引擎处理后实际发送给 API 的提示词 |
| `quality` | VARCHAR | 画质档位：`standard` / `hd` / `master` |
| `style` | VARCHAR | 风格 ID，默认 `"default"`，如 `real`、`travel_guide`、`ccd_snap` 等 |
| `cost_points` | INT | 预扣积分数（standard=5, hd=10, master=15） |
| `points_snapshot` | INT | 实际扣除时的积分快照（用于审计） |
| `image_url` | VARCHAR | 最终图片的腾讯云 COS 链接 |
| `ref_image_url` | TEXT | 图生图参考图链接（支持 Base64 或 COS URL） |
| `parent_id` | INT | 迭代的父任务 ID |
| `root_id` | INT | 迭代链的根任务 ID（用于统计总变体数） |
| `iteration` | INT | 当前迭代次数（0 = 原始首稿） |
| `status` | VARCHAR | `pending` / `generating` / `success` / `failed` |
| `error_msg` | TEXT | 失败原因 |
| `queue_duration` | INT | 排队耗时 (ms) |
| `api_duration` | INT | API 响应耗时 (ms) |
| `generation_duration` | INT | 图片生成耗时 (ms) |
| `storage_duration` | INT | 图片转存 COS 耗时 (ms) |
| `total_duration` | INT | 全链路总耗时 (ms) |
| `share_count` | INT | 用户分享次数（仅计数，无业务逻辑） |
| `created_at` | DATETIME | 任务创建时间（UTC+8） |

## 3. `recharge_logs` (充值/奖励流水表)

> 注意：此表同时记录**付费充值**和**系统奖励**（邀请奖励、PWA 奖励等）。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `user_id` | INT | 外键 → users.id |
| `money_amount` | INT | 实际支付金额（元）；系统奖励时为 0 |
| `amount` | INT | 获得积分数 |
| `screenshot_url` | VARCHAR | 人工报备时的支付截图 COS 链接（在线支付为 NULL） |
| `out_trade_no` | VARCHAR(64) | 商户订单号（在线支付时生成，人工报备时为 NULL） |
| `trade_no` | VARCHAR(64) | 支付平台流水号（支付乐回调后写入）；系统奖励使用 `INVITE_REWARD_*` 等前缀 |
| `payment_method` | VARCHAR(20) | `wxpay` / `alipay`（在线）；NULL（人工/系统） |
| `status` | VARCHAR | `pending`（待审/待支付）/ `success`（到账）/ `rejected`（驳回） |
| `admin_note` | VARCHAR | 管理员审核备注 / 系统奖励说明 |
| `operator_id` | INT | 操作管理员 ID；系统自动奖励时为 0 |
| `created_at` | DATETIME | 创建时间（UTC+8） |

---

## 4. `system_configs` (系统动态配置表) — 重点

后端通过 `backend/core/config.py` 的 `get_config(key)` 函数读取。**缓存有效期 60 秒**，修改数据库后无需重启服务即可生效。

| config_key | 说明 |
| :--- | :--- |
| `OPENAI_API_KEY` | OpenAI / 中转站 API 密钥（核心） |
| `OPENAI_BASE_URL` | API 基础地址，默认 `https://api.openai.com/v1` |
| `DASHSCOPE_API_KEY` | 通义千问提示词润色 API Key |
| `SECRET_KEY` | JWT 签名密钥 |
| `TENCENT_CLOUD_SECRET_ID` | 腾讯云 COS 密钥 ID |
| `TENCENT_CLOUD_SECRET_KEY` | 腾讯云 COS 密钥 Key |
| `TENCENT_CLOUD_COS_BUCKET` | COS 存储桶名称 |
| `TENCENT_CLOUD_COS_REGION` | COS 地域（如 `ap-guangzhou`） |
| `PAYLE_PID` | 支付乐商户 ID |
| `PAYLE_KEY` | 支付乐签名密钥 |
| `PAYLE_API_URL` | 支付乐下单地址 |
| `PAYLE_NOTIFY_URL` | 支付回调通知地址（需外网可达） |
| `PAYLE_RETURN_URL` | 支付完成后跳转地址 |
| `SERVER_IP` | 服务器公网 IP |
| `BT_PANEL_URL` / `BT_PANEL_USER` / `BT_PANEL_PASSWORD` | 宝塔面板访问凭据 |
| `ALIBABA_CLOUD_ACCESS_KEY_ID/SECRET` | 阿里云密钥（备用服务） |
| `DOMAIN_NAME` / `GIT_REPO_URL` | 主域名和 Git 仓库地址 |

## 5. `feedbacks` (用户反馈表)

| :--- | :--- | :--- |

| `content` | TEXT | 反馈内容（必填） |

## 6. `verification_codes` (邮箱验证码表)

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `email` | VARCHAR(255) | 验证码绑定的邮箱 |
| `code` | VARCHAR(10) | 6位随机数字验证码 |
| `expires_at` | DATETIME | 过期时间（创建后 5 分钟） |
| `is_used` | TINYINT(1) | 是否已使用（防重放） |

## 7. `redemption_codes` (兑换码配置表)

| 字段名 | 类型 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | INT | AI | 主键 |
| `code` | VARCHAR(50) | - | 兑换码内容（唯一索引，建议大写） |
| `points` | INT | 50 | 兑换可得积分 |
| `max_uses` | INT | 100 | 发行总量（最多领取人数） |
| `used_count` | INT | 0 | 已领取人数 |
| `is_active` | TINYINT(1) | 1 | 是否启用 |
| `created_at` | DATETIME | 当前时间 | 创建时间 |
| `expired_at` | DATETIME | NULL | 过期时间（可选） |

---

## 8. `redemption_records` (兑换码领取记录表)

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | INT | 主键 |
| `user_id` | INT | 外键 → users.id |
| `code_id` | INT | 外键 → redemption_codes.id |
| `fingerprint` | VARCHAR(100) | 领取时的设备指纹（用于设备级防刷） |
| `created_at` | DATETIME | 领取时间 |

> **索引说明**：`user_id` 与 `code_id` 构成联合唯一索引，确保单用户每个码仅限领取一次。
