# GPT-Image 2 业务与计费核心规范 (V1.3)

本规范定义了 GPT-Image 2 平台的极简计费体系、任务安全锁机制及财务准则。

## 1. 计费核心逻辑 (极简版)
系统统一采用 **1024x1024** 作为默认输出分辨率。

## 2. 任务并发锁与“假死”安抚机制
为了保障服务器性能并防止用户任务堆积，系统实施以下硬性工程约束：

### 2.1 并发限制
- **限制规则**：同一 UID 用户同时只能运行 **2 个** 活动任务。

### 2.2 60s 硬超时逻辑 (Anti-Stuck)
- **触发条件**：后端发起请求后，若 **60 秒** 内未获得 API 响应。
- **动作**：系统自动标记该任务为“超时失败”，**立即释放并发锁，并自动退还预扣积分**。

### 2.3 手动重置入口
- **功能**：在“个人中心/Profile”增加“重置活动任务”按钮。
- **作用**：当用户状态机出现卡死时，允许用户自主一键清空 `pending` 状态。

---

## 3. 财务与退费规则
- **预扣费制**：生成请求下发前锁定积分。
- **失败自动返还**：除“提示词违规”外的系统级失败，一律秒回积分并提示“优先令牌（安抚提示）”。
- **不可退现**：积分永久有效，不支持提现。
