# GPT-Image2 项目交付与待办功能清单

## 1. 项目当前状态 (Status)
- **技术栈**：FastAPI (Python 3.14) + React (Vite) + MySQL。
- **已打通链路**：用户注册(含指纹/IP记录) -> 登录 -> 鉴权守卫 -> 首页生图 -> 积分扣减 -> 管理员充值。
- **安全保障**：全站消费熔断、敏感词过滤、JWT 鉴权。

## 2. 待办功能清单 (Pending Tasks)

### 2.1 前端：缺失页面与模块
- [ ] **“我的创作”历史页**：目前后端 `image_logs` 已有数据，但前端需要一个页面来展示用户过往生成的图片流。
- [ ] **“价格”说明页/弹窗**：需要向用户展示 1/10/30 积分的详细计费规则。
- [ ] **“使用指南”模块**：目前为占位符，需填充具体的 Prompt 技巧或工具使用手册。
- [ ] **个人中心**：目前仅有余额显示，需增加修改密码、查看登录日志（IP/设备）等功能。

### 2.2 核心业务逻辑增强
- [ ] **“图片编辑” (Image Editing)**：前端 UI 已预留 Tab，但后端尚未对接 OpenAI 的 `edit` 或 `variation` 接口（以图生图）。
- [ ] **支付对接**：目前仅有管理员手动充值。建议对接微信/支付宝回调，实现用户自主购买积分。
- [ ] **图片下载优化**：目前下载通过 `<a>` 标签实现，受限于跨域，建议后端增加一个 `Proxy` 接口，通过 `Stream` 强制触发浏览器下载。

### 2.3 技术架构与运维
- [ ] **数据库迁移工具 (Alembic)**：目前使用 `sql/init.sql` 手动维护，建议引入 Alembic 以支持版本化的数据库变更。
- [ ] **Vercel 自动化部署**：需将环境变量配置到 Vercel Dashboard，并验证生产环境下的跨域配置。
- [ ] **指纹算法升级**：目前使用简单的 `userAgent` Base64，建议集成 `FingerprintJS` 以提高唯一性。

---

## 3. 运行指南 (How to Run)
1. 启动命令：根目录下执行 `./start.sh`。
2. 环境变量：查看 `.env` 文件。
3. 数据库：MySQL 库名 `gpt_image_relay`。
