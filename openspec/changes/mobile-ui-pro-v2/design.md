## Context

目前手机端 UI 采用传统的表单提交模式，视觉风格较为落后，且无法支持连续、并行的生图任务流。用户反馈“不够精致”且“交互单一”。本次设计旨在利用 React 的状态驱动优势，构建一个基于“结果堆叠流”的对话式移动端工作站。

## Goals / Non-Goals

**Goals:**
- **对话式结果流**：实现首页的垂直结果卡片堆叠，支持在生图过程中继续下达新指令。
- **大师模式体验对齐**：在移动端完整复现“思维矩阵”动画和“创作笔记”系统。
- **底部抽屉交互 (Bottom Drawer)**：全面替换全屏弹窗，改用手势友好的半屏抽屉进行档位、风格和高级设置。
- **全站视觉升级**：重构历史画廊（双列瀑布流）、会员中心（横向卡片）和个人中心（模块化看板）。

**Non-Goals:**
- 不涉及后端生图接口的逻辑修改（仅前端 UI 适配）。
- 不涉及桌面端 (`HomePage.jsx`) 的功能变更。

## Decisions

- **数据模型：Stack-based Feed**
    - **决策**：在 `MobileHomePage.jsx` 中将单张图片的 state 升级为数组 `historyItems`。
    - **理由**：为了支持“生成一个之后还可以生成第二个”的需求，必须以数组流的形式管理当前会话的结果。
- **交互组件：Sheet/Drawer Pattern**
    - **决策**：使用半透明毛玻璃背景的底部抽屉组件。
    - **理由**：相比全屏弹窗，抽屉在单手操作时体验更佳，且能保持与主画面的空间联系。
- **大师模式反馈：Embedded Matrix**
    - **决策**：将动画逻辑直接嵌入卡片组件。
    - **理由**：在对话流中，每个任务都应有独立的视觉反馈，而非全局的 Loading。

## Risks / Trade-offs

- **[Risk] 渲染开销** → 随着堆叠流变长，大量 GIF/Canvas 粒子动画可能导致卡顿。
  - **Mitigation**：仅对当前正在生成的卡片开启高开销动画，生成完毕后转为静态图，并对历史卡片使用 React 虚拟列表或懒加载。
- **[Risk] 底部导航冲突** → 手机端固定底栏与弹出抽屉的层级冲突。
  - **Mitigation**：设置抽屉的 `z-index` 高于导航栏，并对底部导航区域使用 `pointer-events: none` 穿透处理。
